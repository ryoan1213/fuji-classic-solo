# 富士クラシック 一人予約サイト — 引き継ぎ資料

訪日外国人のお客様に「人がすでに入っている枠」を選んでもらい、一人予約の
キャンセル（前日16:00時点で2名未満だと不成立）を防ぐための公開ページです。

---

## 1. 仕組み（全体像）

```
ValueGolf 一人予約(公開ページ)
        │  fetch_slots.py（ロボ）が読み取る
        ▼
   data.js / data.json   ← 枠データ（日付・時間・海外価格・人数）
        │  index.html が読み込んで表示
        ▼
   公開ページ（お客様が枠を選ぶ）
        │  「Request this group」を押す
        ▼
   現状：お客様のメーラーが開き ryo@preservegolfjapan.com 宛に送信
   （将来：社長のAIプラットフォームに直接送信 → 自動予約に接続予定）
```

## 2. ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | 公開ページ本体（デザイン・表示ロジック） |
| `data.js` / `data.json` | ロボが生成する枠データ。**手で編集しない** |
| `fetch_slots.py` | ロボ。ValueGolfから取得して data.js を生成 |
| `assets/` | 富士クラシックのロゴ・コース写真 |
| `.github/workflows/update.yml` | 自動更新（GitHub Actions のタイマー） |

## 3. ロボが読んでいる場所（重要）

- 一覧（開いている枠の列挙）:
  `https://www.valuegolf.co.jp/one/more_result2.cfm?...&pid=V2LsLwGTUdG4IS&rent_cid=190002801&...`
- 各枠の詳細（**人数・プレー日・海外価格はここが正データ**）:
  `https://www.valuegolf.co.jp/one/plan.cfm?...&oz=<枠ID>`
- ログイン不要（`logon=OFF` の公開ページを使用）
- `pid=V2LsLwGTUdG4IS` は富士クラシックのポータルID。**変わったら fetch_slots.py の先頭の PID を更新**。

### 人数の数え方
詳細ページ「ご予約状況」の **エントリー者の名前ブロック数 = 現在の人数**、
`エントリー受付中` = 空席。`プリザーブゴルフ` という名前があれば**自社予約**と判定し、
データに `ours: true` を付けています（将来「自社の客が2人いる枠だけ確実に安全」と
色分けするための布石）。

### 料金
一覧の価格は**日本居住者向け**。お客様（海外）は対象外なので、詳細ページの
**「海外住居者価格」**（昼食込）を採用。`+¥1,100`＝2名でプレー時の追加料金。
イベント枠（ブリヂストン杯等／海外対象外）は海外価格が無いため**自動で非表示**。

## 4. 自動更新の周期

`.github/workflows/update.yml` の cron で制御：
- 通常：毎時
- 前日午後（締切16:00前後）：30分ごと

変えたい場合はこの cron を編集。手動更新は GitHub の Actions タブ →
"Update Fuji Classic slots" → Run workflow。

## 5. ローカルで動かす場合

```bash
python3 fetch_slots.py     # data.js を更新（追加ライブラリ不要・標準のみ）
# index.html をブラウザで開く
```

## 6. 今後つなぐ予定（社長側）

1. 問い合わせ受信時、AIの初回返信メールに**このページのURLを自動で差し込む**
2. お客様が「Request this group」を押した結果を、メールではなく
   **社長のプラットフォームに直接送信** → AIが自動予約・確定返信

この2点をつなぐと、問い合わせ→確定まで人手ゼロで回ります。
