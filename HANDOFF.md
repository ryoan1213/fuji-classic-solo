# 引き継ぎ・接続マニュアル（社長プラットフォーム連携前提）

このサイトは **単体でも動きますが、最終的には社長のプラットフォームに接続して
「問い合わせ → 枠選択 → 自動予約・自動返信」を人手ゼロで回す**ことを前提に作っています。
このドキュメントは、その**接続に必要な情報**をまとめたものです。

---

## 0. 接続のゴール（全体像）

```
① 問い合わせ受信（富士クラシックWEB → 社長プラットフォーム）
        │
        ▼
② 社長のAIが初回返信メールを生成
        │   ★接続ポイントA：返信メールに「このサイトのURL」を自動で差し込む
        ▼
③ お客様がサイトを開き、人が入っている枠を選んで「Request this group」
        │   ★接続ポイントB：選択結果を社長プラットフォームへ送信（現状はメール）
        ▼
④ 社長のプラットフォームが ValueGolf に自動予約 → 確定メール自動送信
```

このサイトが担うのは **②③の「お客様に見せて選ばせる」部分**。
**接続ポイントA・Bの2か所**を社長プラットフォーム側で繋げば一気通貫になります。

---

## 1. データ契約（接続の要）— `data.json`

社長プラットフォームがこのサイトのデータを使う場合、参照するのはこれだけです。
公開URL: `https://ryoan1213.github.io/fuji-classic-solo/data.json`（CORS可・認証不要）

```jsonc
{
  "generatedAt": "2026-06-01 19:12 JST",   // 最終更新（日本時間）
  "course": "Fuji Classic",
  "priceNote": "Overseas-golfer rate, lunch included.",
  "slots": [
    {
      "id": "T136221690",      // ValueGolf の枠ID（oz）。予約APIのキーに使える
      "date": "2026-06-03",    // プレー日(ISO)。確定日時+1日で算出
      "weekday": "Wed",
      "io": "IN",              // IN / OUT
      "time": "08:35",         // ティータイム
      "minPlayers": 2,         // 最低催行人数
      "maxSeats": 4,           // 1組の上限
      "overseasPrice": 33800,  // ★海外ゴルファー価格(昼食込)。お客様が払う額
      "surcharge2p": 1100,     // 2名でプレー時の追加料金
      "players": 1,            // 現在の予約人数（正データ＝詳細ページから取得）
      "ours": true,            // この組に自社予約(プリザーブゴルフ)が入っているか
      "deadline": "06/02 16:00"// 確定締切（前日16:00）
    }
  ]
}
```

> このJSONを社長プラットフォームがそのまま読めば、空き枠・人数・正しい料金が全部手に入ります。

## 3. ★接続ポイントB：「Request this group」を社長プラットフォームへ送る

### 現状
`index.html` の `mailto()` 関数で、選択内容を `ryo@preservegolfjapan.com` 宛の
**メール下書き**として開きます（サーバー不要で動く暫定仕様）。

### 接続後（社長プラットフォーム側）
ボタン押下時に、社長のエンドポイントへ **POST** する形に差し替えます。
変更箇所は `index.html` の `mailto()` / `card()` 内のボタン部分のみ。送るデータ例：

```jsonc
POST https://<社長のエンドポイント>/solo-booking
{
  "slotId": "T136221690",     // data.json の id をそのまま
  "date": "2026-06-03",
  "io": "IN",
  "time": "08:35",
  "overseasPrice": 33800,
  "customer": { "name": "", "email": "", "country": "" }  // フォームで取得
}
```

※現状ページはお客様情報フォームを持たないため、POST化する場合は
　簡単な入力フォーム（氏名・メール）を足す想定。ここは社長側の仕様に合わせて調整します。

## 4. データの取得元（ValueGolf）と「正データ」の所在

ロボ `fetch_slots.py` が読んでいる場所。社長側で独自に取り込む場合も同じ手順です。

- **一覧（枠IDの列挙のみに使用）**
  `GET https://www.valuegolf.co.jp/one/more_result2.cfm?pid=V2LsLwGTUdG4IS&rent_cid=190002801&exh_flg=V&pageRows=3000&pageNo=0&openGroupFlg=1&prm2z=5`
  → HTMLから `oz=T...` を抜き出すだけ。**※一覧は特殊プランで日付が混ざるため、日付・人数は信用しない。**

- **詳細（★ここが正データ）**
  `GET https://www.valuegolf.co.jp/one/plan.cfm?pid=V2LsLwGTUdG4IS&rent_cid=190002801&oz=<枠ID>&exh_flg=V`
  - **プレー日** = 「ご予約状況」内の確定日時 `「YYYY/MM/DD(曜) 16:00」` ＋1日
  - **人数** = 「ご予約状況」のエントリー者の名前ブロック数（`エントリー受付中`＝空席）
  - **自社判定** = エントリー者に `プリザーブゴルフ` がいれば `ours: true`
  - **海外価格** = `海外住居者価格：￥XXXXX`（日本語表記を優先。英語表記とズレる枠あり）
  - **イベント枠**（ブリヂストン杯等）= 海外価格が無い → **自動で非表示**

- ログイン不要（`logon=OFF` の公開ページ）
- `pid=V2LsLwGTUdG4IS` は富士クラシックのポータルID。**変わったら `fetch_slots.py` 先頭の `PID` を更新**

## 5. ロボと自動更新（運用）

- `fetch_slots.py`：Python標準ライブラリのみ（追加インストール不要）
- `.github/workflows/update.yml`：GitHub Actions のタイマー
  - 通常：毎時 / 締切前（JST午後）：30分ごと（cronで調整可）
  - 手動実行：GitHubの Actions タブ → "Update Fuji Classic slots" → Run workflow
- クラウドで動くのでPC起動不要。`data.js` / `data.json` を自動更新 → サイトに反映

---

## 6. 社長プラットフォームへの接続パターン（3択）

| パターン | 内容 | 向いている場合 |
|---|---|---|
| **① リンクするだけ** | このサイトのURLを初回メールに貼るだけ。選択結果はメールで受ける | まず最速で運用開始したい |
| **② data.json を取り込む** | 社長プラットフォームが `data.json` を読み、独自UIで表示・予約連携 | プラットフォーム内に統合したい |
| **③ ロジックを取り込む** | `fetch_slots.py` の取得ロジックを社長のバックエンドに移植 | 完全に自社管理したい |

どのパターンでも、**4章のデータ取得方法**と**1章のデータ契約**がそのまま使えます。

---

## 7. 注意点

- `pid` が変わったら `fetch_slots.py` の `PID` を更新（4章）
- 取得上限は3000枠（現状160枚程度なので余裕。将来さらに増えても取りこぼさない）
- 表示時刻は日本時間(JST)。`generatedAt` が「いつ時点の最新か」を示す
- イベント枠（海外対象外）は自動除外。除外件数はサイト下部に注記表示
